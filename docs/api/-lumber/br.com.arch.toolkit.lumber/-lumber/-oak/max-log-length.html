maxLogLength

open fun maxLogLength(maxLogLength: Int): Lumber.Oak(source)

Sets a one-time max log length to be used for the next logging call on this specific Oak. When enabled, some loggers might skip logging the message based on their implementation. The quiet flag is temporary and only affects the immediate next log message.

The flag is stored using a ThreadSafe to ensure it is only applied for the current thread and is cleared automatically after the log call.

Return

The Oak instance for method chaining.

Example:

Lumber.maxLogLength(10).tag("Tag").debug("Debug message");
// Expected output:
Debug: [Tag #0] Debug mess
Debug: [Tag #1] age
